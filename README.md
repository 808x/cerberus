This repo merges your two existing pieces of work into one runnable,
explainable, hackathon-ready system, and **fills in the piece that was
missing: Task 1, the trained anti-spoofing classifier.**

## 1. Problem statement, distilled

From your slides + doc:

- **Goal:** build a system that (a) clones voices, (b) detects AI-generated
  speech, (c) reasons autonomously about spoofing threats, (d) stays robust
  to noise and unseen attacks, (e) explains its verdicts in real time.
- **Required models/tools called out in the problem statement:**
  - Voice cloning: **XTTS**, Coqui TTS, FishSpeech
  - Anti-spoofing classifier: **ECAPA-TDNN, AASIST, RawNet2, WavLM**
  - Features: **MFCC, Spectrogram, SSL embeddings**
  - Agentic framework: **LangGraph** (LangChain/CrewAI/AutoGen as alternatives)
  - Audio processing: **torchaudio, librosa, SpeechBrain, Hugging Face Transformers**
  - Frontend: **Streamlit or Gradio**
  - Evaluation: Accuracy, Precision, Recall, F1, ROC-AUC, **EER**; robustness to
    noise and unseen attacks
- **Evaluation rubric weights:** cloning quality, classifier metrics, agentic
  reasoning/tool-use/explainability, robustness, engineering quality
  (latency, code quality, reproducibility), and an "X-factor" for novel
  agent design.

This build satisfies every deliverable in the "Final Output" slide: two
classifiers (see `--mode quick` vs `--mode xtts` below = your "Run 1" and
"Run 2"), a paired real/synthetic dataset, an agentic workflow, clean vs
noisy evaluation with metrics/plots, a working local demo, and a
reproducible codebase with a manifest of dependencies.

### Note on model choice for Task 1

The problem statement lists AASIST/RawNet2/WavLM as *possible* models — those
are end-to-end deep nets over raw waveform/SSL embeddings, and typically need
large labelled corpora (ASVspoof-scale) plus real GPU training time to beat a
simpler baseline. Given a hackathon timeline, Task 1 here is a compact **MLP
over MFCC statistics** (the same "MFCC" feature family explicitly listed),
trained on data generated by your own Task 0 cloner — so it's directly
relevant to the attacks you're actually demonstrating, trains in minutes on
CPU, and is fully swappable later (see `src/task1_classifier/model.py`) for
an ECAPA/AASIST-based model if you have GPU budget and more data.

## 4. External requirements (things you need to add/install yourself)

| Requirement | Needed for | Notes |
|---|---|---|
| Python 3.10+ | everything | matches problem statement |
| `pip install -r requirements.txt` | everything | see file for the full list |
| GPU (≥8GB VRAM recommended, not required) | `--mode xtts` cloning, faster training | CPU works fine for `--mode quick` and for Task 1 training (it's a small MLP) |
| `pip install TTS speechbrain` | only if using `--mode xtts` in Task 0 | commented out in `requirements.txt` by default — first run downloads ~2GB of XTTS weights |
| Internet access (first run only) | CMU ARCTIC auto-download, XTTS weights, SpeechBrain ECAPA weights | after first download everything runs offline |
| `ANTHROPIC_API_KEY` or `OPENAI_API_KEY` | only if you want real free-text LLM explanations instead of the deterministic stub | set `llm.provider` in `config.yaml` and put the key in `.env` |
| VS Code + Python extension | for the `.vscode/launch.json` one-click Run/Debug configs | optional — everything also runs from a plain terminal |

Nothing else is required to get a fully working, explainable end-to-end demo — the default config (`llm.provider: stub`, `--mode quick` data generation) needs **zero API keys and zero large downloads.**

## 5. How to run — step by step (VS Code or terminal)

### Step 0 — Open in VS Code
Open the `cerberus_antispoofing/` folder in VS Code. Select/create a Python 3.10+ interpreter (`Ctrl+Shift+P` → "Python: Select Interpreter"). A `.venv` is recommended:

```bash
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### Step 1 — Generate the adversarial dataset (Task 0)
Uses the team recordings in `../cbs_adi/Data/Real/` and XTTS clones in `../cbs_adi/Data/Clones/`:
```bash
python src/task0_cloning/build_adversarial_dataset.py --clear --n-distort-per-real 2
```
This creates:
  - `data/real/` — clean real chunks
  - `data/fake_synth/` — XTTS synthetic chunks
  - `data/fake_distort/` — real chunks with adversarial distortions

(The legacy `generate_dataset.py` script still exists for CMU ARCTIC if needed.)

*(In VS Code: Run and Debug panel → no launch config needed here, just run from the integrated terminal, or right-click the file → "Run Python File in Terminal".)*

### Step 2 — Train the anti-spoofing classifier (Task 1)
```bash
python src/task1_classifier/train.py
```
Or in VS Code: **Run and Debug → "Task1: Train classifier"**.
This prints per-epoch accuracy/F1/AUC and saves:
- `models/task1_classifier.pt` (weights + scaler + config)
- `models/task1_scaler.json`

### Step 3 — Evaluate it properly (Accuracy/Precision/Recall/F1/ROC-AUC/EER)
```bash
python src/task1_classifier/evaluate.py
```
Or **Run and Debug → "Task1: Evaluate classifier"**.
Outputs `reports/task1_eval_metrics.json`, `reports/roc_curve.png`, `reports/confusion_matrix.png`.

### Step 4 — Run the full agent on one file (Task 2)
```bash
python src/task2_agent/graph.py samples/test_sample.wav
```
Or **Run and Debug → "Task2: Run agent on sample"**.
You'll get a JSON verdict with `verdict`, `risk_level`, `confidence`,
`attack_type_guess`, `explanation`, and the full `reasoning_trace` showing
exactly which tools the agent decided to call and why.

Try it on the bundled team samples:
```bash
python src/task2_agent/graph.py samples/team_real.wav
python src/task2_agent/graph.py samples/team_fake_synth.wav
python src/task2_agent/graph.py samples/team_fake_distort.wav
```

### Step 5 — Robustness evaluation (Task 3)
```bash
python src/task3_robustness/robustness_eval.py --data-root data --checkpoint models/task1_classifier.pt
python src/task3_robustness/plot_robustness.py --report reports/task3_robustness.json
```
Produces:
  - `reports/task3_robustness.json` — accuracy/F1 under noise, reverb, compression,
    band-pass filtering, pitch shift, time stretch, and feature-space adversarial noise
  - `reports/task3_robustness.png` — summary bar chart

The agent's robustness context tool reads this report to adjust confidence/risk
for incoming samples.

### Step 6 — Launch the dashboard (Task 4)
```bash
python -m uvicorn src.api.main:app --host 0.0.0.0 --port 8000
```
Then open `http://localhost:8000` in your browser.

The dashboard provides:
  - **Audio input**: upload a `.wav`, use your microphone, or pick a bundled sample
  - **Live probability stream**: microphone audio is chunked into ~3.5 s windows and the classifier probabilities are plotted in real time
  - **Agent verdict + explanation**: full Task 2 reasoning, risk level, and recommended action
  - **Pipeline buttons**: run Task 0–3 directly from the UI with live terminal output
  - **Technical dashboard**: expandable panel with terminal logs, extracted features, reasoning trace, and raw JSON

### All-in-one
```bash
bash scripts/run_pipeline.sh quick    # or: bash scripts/run_pipeline.sh xtts
```

## 6. How to test that it's actually working

1. **Sanity check the mock fallback is gone:** run Step 4 before Step 2 — you should see `"model_used": "mock_heuristic"` in the output. After Step 2, the same command should show `"model_used": "task1_trained_mlp"`. That confirms the trained model is being picked up.
2. **Check discrimination ability:** run the agent on `samples/real_sample_cloner_source.wav` (should trend toward `REAL`/`LIKELY_REAL`) vs. `samples/fake_sample_xtts_clone.wav` (should trend toward `FAKE`/`LIKELY_FAKE`). These are a genuine matched real/cloned pair from your own XTTS pipeline.
3. **Check the metrics file:** `reports/task1_eval_metrics.json` should show accuracy/F1/ROC-AUC meaningfully above chance (>0.5) and EER meaningfully below 0.5. If it's near chance, generate more training data (`--n-per-speaker` higher) or add more speakers.
4. **Check robustness degrades gracefully, not catastrophically:** in `reports/task3_robustness.json`, accuracy should drop as SNR decreases (that's expected and worth discussing in your writeup) but shouldn't collapse to random guessing at 20dB/10dB.
5. **Check explainability end-to-end:** every response from `graph.run()` includes a non-empty `reasoning_trace` list — this is your evidence for the "Explainability" rubric line.
6. **Latency check (Engineering Quality rubric line):** time a single inference:
   ```bash
   time python src/task2_agent/graph.py samples/test_sample.wav
   ```
   With `--mode quick` MFCC features + a small MLP, a single clip should classify in well under a second on CPU (excluding one-time model load).

## 7. Swap-in points for going further

- **Better classifier:** replace `src/task1_classifier/model.py` with a CNN over log-mel spectrograms, or wrap a pretrained SpeechBrain ECAPA-TDNN / WavLM embedding + linear head. Keep `forward(x) -> logits` and update `classifier_tool.py`'s feature vector accordingly.
- **Real LLM reasoning:** set `llm.provider: anthropic` (or `openai`/`ollama`) in `config.yaml`, put your key in `.env`, `pip install anthropic` (or `openai`). No other code changes needed — `agents/llm_interface.py` already has the call implemented.
- **More attack diversity:** add more entries to `src/task2_agent/data/known_attack_signatures.json` as Task 3 characterizes new attack feature ranges — the agent's `known_attack_signature_check` tool will pick them up automatically.
- **Live microphone / deepfake video / multi-language:** listed in your slides as future enhancements — not implemented here, but the modular tool structure (everything is a small function with a stable input/output contract) is built to make each of these additive rather than a rewrite.
